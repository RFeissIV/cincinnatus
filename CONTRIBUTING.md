# Contributing to Cincinnatus

Thank you for your interest in contributing to Cincinnatus.

This project was built by a plant scientist to bridge siloed scientific domains. Input from computer scientists, mathematicians, bioinformaticians, and domain experts is especially valued — there are aspects of this system that someone in those arenas will identify immediately as improvable, and that feedback is exactly what this project needs.

## How to contribute

### Report issues
- Entity matching failures (e.g., "my chemical wasn't found")
- Incorrect paths or provenance
- Installation problems
- Performance issues
- Scientific inaccuracies in results

### Request databases
Open an issue with:
- Database name and URL
- What domain it covers
- Why it would be valuable for cross-domain reasoning
- Whether the data is freely available

### Suggest algorithmic improvements
The author is not a computer scientist. If you see a better algorithm, data structure, or approach — please suggest it. Ideas for v2 are actively sought, including:
- Graph traversal optimizations
- Learned scoring or traversal policies
- Better entity resolution approaches
- Evaluation frameworks and benchmarks

### Submit code
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request with a clear description

### Improve documentation
- Tutorials for specific scientific domains
- Installation guides for different platforms
- Example queries and expected outputs

## Code style
- Python 3.8+ compatible
- Type hints where practical
- Docstrings for public methods

## Attribution
Contributors who make substantial additions (new databases, major features, algorithmic improvements) will be offered co-authorship on the v2 publication. All contributors will be acknowledged in the README.

## Questions
Open an issue or contact the author.
